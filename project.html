Project goals

The primary goal is to create a FOSS replacement for Stack Overflow Chat. The secondary goal is to create a reusable web multi-user chat solution based on XMPP.

The motivation for starting this project is twofold. First, many implemented features are plain broken, as time and experience has proven:

  • The global community moderation (a.k.a. flagging) system does not work properly. Not in the sense that flagging does nothing, but in the sense that flagging is an extremely delayed response (unless we coordinate flagging, which is probably against the mighty rules) to an extremely time-sensitive situation. We've had cases of people getting banned for something said MONTHS ago, but we have trouble getting and keeping trolls annoying us RIGHT NOW out. It just doesn't fit.
  • Room ownership is useless, and mostly for the show. They call us "owners", but we can't even enforce simple rules that ensure good chatting experience for the community. Oh but yeah, we can change the topic. And there's that nonsensical ability to move messages between rooms, that's used pretty much exclusively as a workaround for lack of ability to delete messages.
  • Chat offers Markdown formatting, but in a reduced version, for no other reason than "we say you won't need that".
  • Mobile interface is just ridiculously under-featured.
  • Oneboxing (inlining external resources) has no opt-out, and sometimes disrupts the chat flow to the point of covering an entire discussion.
  • Chat stores history, but the search engine is close to being useless.
  • And the protocol is closed and undocumented, so we can't easily replace the client.

And we get to the second reason: neglect. We (the chat users) have reported these, but the reports either get no response, or are declined on the grounds of being incompatible with SO policies/vision for the chat. A vision that's detached from reality and what users actually want. The ideal chat room would be full of mindless drones droning about insert technology we happened to include in the room title 24/7.

All of these add up to chat experience that gets extremely tiresome and unpleasant. Thankfully, despite SO's best efforts, we managed to form a community that transcends the technology. And that's how we arrive at Lounge<Chat>.

Why XMPP?

We've decided to use XMPP, because it's a standard, open protocol. There are many mature server and client implementations we can leverage to reduce the amount of work needed to get this project going. And at the end, once we flesh out the details, we hope to create a formal extension protocol, so that more clients and servers can be implemented with the chat features we came to like so much.
